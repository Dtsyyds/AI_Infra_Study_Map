## 题解

- 暴力解法

一种是横向查找，直接查找两两之间的最小公共前缀，然后就可以得到结果；
另一种是纵向查找，挨个儿字符遍历比较，直到找到不同的字符为止。

- 巧妙思路

直接借助sort排序的规则，将字符串数组排序后，直接比较第一个和最后一个字符串即可。

1. 逐字符比较（核心规则）
从左到右逐个比较字符串中相同位置的字符，比较依据是字符的底层数值（char 类型的整数值，通常为 ASCII 或 UTF-8 编码）。

一旦遇到两个不同的字符，数值较小的字符所在的字符串被判定为“更小”，排在前面。

例如："flight" vs "flow"，前两个字符 f、l 都相同，第三个字符 i（ASCII 105） < o（ASCII 111），所以 "flight" 排在 "flow" 前面。

2. 前缀规则（短字符串优先）
如果两个字符串在较短的字符串长度内所有字符都完全相同（即一个是另一个的前缀），则较短的字符串排在前面。

例如："flow" 是 "flower" 的前缀，所以 "flow" < "flower"。

3. 不依赖本地化（Locale）
默认的 operator< 是基于字符的二进制值进行比较的，不受系统区域语言设置（如中文、英文排序习惯）的影响。它不会将 A 和 a 视为相等，也不会处理变音符号（如 é），完全按码点值排序（大写字母 A-Z 的 ASCII 值 65-90 小于小写字母 a-z 的 97-122，因此所有大写字母会排在小写字母之前）。
