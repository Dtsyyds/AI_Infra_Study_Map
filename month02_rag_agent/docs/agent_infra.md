# Agent Infra 简介

## Agent Runtime
Agent Runtime 是基于 python 开发的一个工程化实例。用户输入进入 LLMAgent 后，系统结合默认prompt 结合用户输入构建输出给 LLM 的 prompt,调用 FakeLLM 或真实模型，从模型输出中提取并解析 action，再通过 Executor 调用受沙盒保护的工具，工具 Observation 会写回当前任务 Memory ，驱动下一轮推理， 直到 Finish。

## 工具执行
工具执行主要是对 agent 输出决策的响应，执行具体的工具，并返回执行结果。在执行前会检查所需要的工具是否在注册表中，提供的参数是否符合要求，以及执行的目标是否在沙盒中。

## 沙盒安全
沙盒主要是用于隔离工具的执行环境，避免工具执行过程中的危险操作，比如敏感文件读取，跳出当前工作空间操作等危险操作。在执行前会先检查输入路径是否安全，后续也在 Prompt 中做进一步的限制，最终在工具执行时再做一层保障，保证执行工具的安全性。 

## Trace 与评测
Trace 是为了验证系统的完整性。记录了用户的每一步请求，以及 agent 对该请求做出的决策和调用对应工具获得的执行结果。方便后续的评测和分析。

## memory
目前 memory 只实现了最简单的短期记忆存储，保存用户输入，agent 输出，工具 observation。同时也有展示记忆和删除清空记忆等功能的实现。

## eval
eval 提供 agent 工具的测评，提供基础的测试案例，通过多轮用 全新的 agent 实例进行测试，并记录结果。并对运行结果用指标进行评估。
